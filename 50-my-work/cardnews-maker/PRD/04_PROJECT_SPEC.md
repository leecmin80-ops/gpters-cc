# cardnews-maker -- 프로젝트 스펙

> AI가 스킬을 만들고 실행할 때 지켜야 할 규칙과 절대 하면 안 되는 것.
> 이 문서를 AI에게 항상 함께 공유하세요.

---

## 기술 스택

| 영역 | 선택 | 이유 |
|------|------|------|
| 런타임 | Claude Code Skill | 스킬 형태로 반복 실행 가능 |
| 기획서 출력 | Markdown (.md) | 읽기/수정 쉽고 버전 관리 가능 |
| 카드뉴스 출력 | HTML + CSS | 별도 도구 없이 브라우저에서 바로 확인 |
| 폰트 | Noto Sans KR (Google Fonts) | 한글 가독성 최고, 무료 |
| 카드 규격 | 1080x1080px | 인스타그램 정사각 피드 표준 |

## 절대 하지 마 (DO NOT)

> AI에게 코드를 시킬 때 이 목록을 반드시 함께 공유하세요.

- [ ] 카드 장수를 10장이 아닌 다른 수로 바꾸지 마 (Phase 1 고정)
- [ ] 특정 주제(예: 하이닉스)에 종속된 스킬로 만들지 마 — 주제는 항상 입력값
- [ ] 투자 권유, 매수·매도 판단, 수익 보장 표현을 넣지 마
- [ ] 외부 JavaScript 라이브러리를 HTML에 추가하지 마 — 순수 HTML+CSS만
- [ ] 이미지 파일을 외부에서 불러오지 마 — 이모지와 CSS로만 시각 표현
- [ ] 기획서 없이 HTML을 바로 생성하지 마 — 항상 기획서 → HTML 순서
- [ ] 사용자 파일을 임의로 삭제하거나 덮어쓰지 마 — 확인 먼저

---

## 항상 해 (ALWAYS DO)

- [ ] 주제를 입력받는 것으로 시작
- [ ] 기획서의 카드 역할 배분: Hook 1장, 본문 7장, 정리 1장, CTA 1장
- [ ] HTML 카드는 1080x1080 정사각, 세로 스크롤 배치
- [ ] Noto Sans KR 폰트 사용 (Google Fonts CDN)
- [ ] 모바일에서도 확인 가능하도록 반응형 처리
- [ ] 카드 번호(n/10) 표시
- [ ] 생성 완료 후 검토 체크리스트 실행
- [ ] 결과물을 `50-my-work/cardnews-maker/`에 저장

---

## 검토 체크리스트 항목

| 항목 | 확인 기준 |
|------|----------|
| Hook | 1번 카드에 호기심 유발 요소가 있는가 |
| 본문 분량 | 카드당 2~3줄 이내인가 |
| 역할 배분 | Hook 1 / 본문 7 / 정리 1 / CTA 1 구성인가 |
| CTA | 마지막 카드에 행동 유도 문구가 있는가 |
| 시각 요소 | 각 카드에 이모지 또는 시각 표현이 있는가 |
| 테마 일관성 | 선택한 테마의 색상이 전체에 일관되게 적용됐는가 |
| 모바일 | 작은 화면에서도 텍스트가 읽히는가 |

## 내장 테마 스펙

### mint (기본)
- 메인: #00b894, 배경: #e0f7f1 → #ffffff, 텍스트: #1a1a1a
- 느낌: 깨끗하고 시원한

### warm
- 메인: #e17055, 배경: #fff4e6 → #ffffff, 텍스트: #2d3436
- 느낌: 따뜻하고 친근한

### dark
- 메인: #dfe6e9, 배경: #2d3436 → #636e72, 텍스트: #ffffff
- 느낌: 세련되고 모던한
