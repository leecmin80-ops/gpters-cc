# cardnews-maker -- Phase 분리 계획

> 한 번에 다 만들면 복잡해져서 품질이 떨어집니다.
> Phase별로 나눠서 각각 "진짜 동작하는 스킬"을 만듭니다.

---

## Phase 1: MVP

### 목표
주제 입력만으로 인스타그램 카드뉴스 기획서 + HTML이 자동 생성된다

### 기능
- [ ] 주제 텍스트 입력 → 10장 기획서 자동 생성
- [ ] 기획서 기반 1080x1080 HTML 카드뉴스 생성
- [ ] 검토 체크리스트 (텍스트 분량, 역할 배분, CTA 포함 여부)
- [ ] 디자인 테마 3종 내장 (mint, warm, dark)
- [ ] `50-my-work/cardnews-maker/`에 자동 저장

### 데이터
- Input (topic, theme)
- Plan (card_no, role, title, body, visual, design_note)
- Theme (name, colors, font_style)

### "진짜 스킬" 체크리스트
- [ ] 주제만 바꿔서 다시 실행하면 새 카드뉴스가 나옴
- [ ] 3종 테마가 각각 다른 디자인으로 적용됨
- [ ] 검토 체크리스트가 누락 항목을 잡아냄
- [ ] 브라우저에서 HTML을 열면 1080x1080 카드가 세로로 나열됨

### Phase 1 시작 프롬프트
```
이 PRD를 읽고 Phase 1을 구현해주세요.
@PRD/01_PRD.md
@PRD/02_DATA_MODEL.md
@PRD/04_PROJECT_SPEC.md

Phase 1 범위:
- 주제 입력 → 10장 기획서 생성
- 기획서 → HTML 카드뉴스 생성 (1080x1080, 세로 스크롤)
- 검토 체크리스트 (텍스트 분량, 역할 배분, CTA)
- 디자인 테마 3종 (mint, warm, dark)
- 50-my-work/cardnews-maker/ 자동 저장

반드시 지켜야 할 것:
- 04_PROJECT_SPEC.md의 "절대 하지 마" 목록 준수
- 10장 고정 (Hook 1 / 본문 7 / 정리 1 / CTA 1)
- 다른 주제로 바꿔도 같은 품질로 재생성 가능
```

## Phase 2: 확장

### 전제 조건
- Phase 1이 안정적으로 동작하는 상태

### 목표
입력 방식을 다양화하고 카드 장수를 유연하게 조절

### 기능
- [ ] URL/파일 입력 지원 (주제 대신 자료 직접 투입)
- [ ] 카드뉴스 장수 선택 (5/8/10/15장)
- [ ] 브라우저 자동 열기 + 스크린샷

### 추가 데이터
- Input에 source_type 필드 추가 (text / url / file)
- Plan에 total_cards 필드 추가

### 통합 테스트
- Phase 1 기능 (주제 텍스트 입력 + 10장 생성)이 여전히 정상 동작하는지 확인

---

## Phase 3: 고도화

### 전제 조건
- Phase 1 + 2가 안정적으로 동작 중

### 목표
완성도 높은 시각 결과물과 배포용 파일 생성

### 기능
- [ ] 카드별 AI 이미지 자동 생성
- [ ] PDF 내보내기
- [ ] 인스타 업로드용 개별 이미지 분리 저장 (카드당 1장 PNG)

### 주의사항
- AI 이미지 생성은 외부 API 비용 발생 가능
- PDF 변환은 추가 라이브러리 필요
