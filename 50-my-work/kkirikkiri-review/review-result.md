## cardnews-maker 리뷰

### 관점별 핵심 평가

| 관점 | 평가 |
|------|------|
| 콘텐츠 기획자 | 10장 고정 구조는 일반적인 인스타 포맷에 맞지만, 단순 주제는 늘어지고 복잡한 주제는 압축이 심해진다 |
| 카피라이터 | 2~3줄 제약은 적절하나 톤 가이드가 없어 매번 문체가 달라질 수 있다. CTA 목표 행동 선택이 없다 |
| 디자이너 | CSS 변수 기반 테마 관리는 깔끔하나 이모지 의존 시각 표현은 OS별 렌더링 차이 리스크. 텍스트 오버플로우 방어 없음 |
| QA 리뷰어 | 이어하기/덮어쓰기 안전장치는 좋으나 체크리스트 판단 기준이 수치화되지 않아 형식적 자기선언 위험 |

### 개선안 TOP 3 (바로 고칠 것)

| # | 문제 | 제안 |
|---|------|------|
| 1 | 체크리스트 판단 기준 불명확 | 카드당 글자 수 80자 이하=pass, 81~120=warn, 121+=fail 등 수치 기준 추가 |
| 2 | 톤 가이드 없음 | Step 1에 "친근한/전문적인/짧고 강한" 3종 톤 옵션 추가 |
| 3 | 텍스트 오버플로우 방어 없음 | HTML 생성 규칙에 카드당 최대 글자 수 + CSS clamp() 규칙 추가 |

### 다음에 고칠 것

- CTA 유형 선택 (팔로우/저장/링크클릭)

| 데이터 정리자 | 중복 정리 기준이 "중복·유사"로만 정의되어 실행마다 결과가 달라질 수 있음. CSV 컬럼 구조 미명세 |
| 리스크 검토자 | Disclaimer 존재는 긍정적이나 "Analyst View", "Bull/Bear Case" 용어가 증권사 리포트와 동일해 오인 여지 |

| 2 | 중복 정리 기준 불명확 | "같은 날 같은 사건, 제목 공통 키워드 2개 이상 = 동일 이슈" 규칙 명시 |
| 3 | Bull/Bear 이분법 한계 | "긍정 요인 / 부정 요인 / 주목할 변수" 3분류로 변경 |

## 두 스킬 공통 개선 원칙

| 원칙 | 내용 |
|------|------|
| 판단 기준을 수치화하라 | "좋다/나쁘다"가 아니라 글자 수, 기사 수, 이슈 수 등 숫자로 pass/fail 기준을 정해야 일관성이 유지된다 |
| 선택지를 미리 제시하라 | 톤, CTA 유형, 수집 건수 등 가변 요소는 사용자에게 선택지를 먼저 보여주고, 기본값도 명시해야 한다 |
| 용어가 오해를 부르지 않게 하라 | "Analyst View", "Bull/Bear Case" 같은 전문 용어는 도구 성격에 맞게 순화하거나 AI 생성임을 명시해야 한다 |
| fallback 결과를 같은 포맷으로 통일하라 | MCP/WebSearch 등 소스가 달라도 다음 단계에 넘기는 데이터 포맷은 동일해야 파이프라인이 안정적이다 |
| 고정값에 "왜 이 값인지" 근거를 두라 | 10장, 30건, 5~7개 이슈 같은 고정값은 향후 변경 시 판단 기준이 있어야 한다 |
