# stock-news-analyst -- PRD (Product Requirements Document)

> 생성일: 2026-05-24
> 생성 도구: Show Me The PRD

---

## 1. 제품 개요

### 한 줄 요약
종목명 또는 키워드를 입력하면 뉴스 수집 → 중복 정리 → 월가 스타일 종목 분석 리포트를 자동 생성하는 Claude Code 스킬

### 해결하는 문제
종목 관련 뉴스를 매번 수동으로 검색하고, 중복 기사를 걸러내고, 이슈별로 분류해서 분석하는 반복 작업의 비효율

### 핵심 가치
키워드 하나로 뉴스 수집부터 애널리스트급 분석 리포트까지 일관된 구조로 자동 생성. 어떤 종목/키워드든 재사용 가능.

---

## 2. 사용자

### 주요 사용자
- **누구**: 본인 (개인 사용)
- **상황**: 특정 종목이나 산업 키워드의 최신 뉴스 흐름을 파악하고 싶을 때
- **목표**: 뉴스 기반으로 종목의 현재 상황을 빠르게 정리하고 판단 근거를 확보

### 사용자 시나리오
1. Claude Code에서 stock-news-analyst 스킬을 호출하고
2. 키워드(예: "삼성전자", "2차전지", "엔비디아")를 입력하면
3. 뉴스 수집 → 중복 정리 → 분석 리포트가 자동 생성되어 저장됨

---

## 3. 핵심 기능

| 기능 | 설명 | 우선순위 | 복잡도 |
|------|------|----------|--------|
| 뉴스 수집 | 네이버 뉴스 API로 최신 기사 30건 수집 | P1 (MVP) | 보통 |
| 중복 정리 | 같은 사건/이슈를 다룬 기사를 묶어 정리 | P1 (MVP) | 보통 |
| 분석 리포트 | 이슈별 분석, Bull/Bear Case, 모니터링 지표 | P1 (MVP) | 보통 |
| 수집 건수 선택 | 15/30/50건 중 선택 | P2 | 간단 |

---

## 4. 사용자 흐름

### 핵심 흐름
```
키워드 입력 -> 뉴스 수집(30건) -> 중복 정리 -> 분석 리포트 -> 저장 완료
```

### 상세 흐름
1. **키워드 입력**: 종목명 또는 산업 키워드 입력
2. **뉴스 수집**: NaverSearch MCP로 최신 뉴스 30건 수집 → `01-news-urls.csv`
3. **중복 정리**: 같은 이슈 기사 묶음 → `02-deduped-news.md`
4. **분석 리포트**: 월가 스타일 리포트 → `03-stock-news-report.md`
5. **후속 안내**: 후속 검색 키워드 5개 + 모니터링 지표 제시

---

## 5. 성공 기준

- [ ] 키워드만 입력하면 3개 파일이 자동 생성된다
- [ ] 30건 뉴스에서 중복을 제거하고 5~7개 이슈로 묶인다
- [ ] 분석 리포트에 Executive Summary, Bull/Bear Case, Key Metrics가 포함된다
- [ ] 다른 종목으로 바꿔도 같은 품질로 재생성된다
- [ ] 투자 권유가 아니라는 Disclaimer가 반드시 포함된다

---

## 6. 안 만드는 것 (Out of Scope)

- 실시간 주가 데이터 연동 -- 이유: MVP는 뉴스 분석에 집중
- 매수·매도 추천 -- 이유: 투자 자문 금지
- 기사 본문 전체 크롤링 -- 이유: 제목+요약으로 충분, 저작권 리스크
- 정기 자동 실행 -- 이유: Phase 2에서 스케줄 기능으로 확장
