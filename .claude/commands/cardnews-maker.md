# cardnews-maker

> 주제를 입력하면 인스타그램 카드뉴스 기획서와 HTML 카드뉴스를 자동 생성하는 스킬

---

## 설정

| 설정 | 기본값 | 변경 방법 |
|------|--------|-----------|
| 테마 | mint | 실행 시 mint / warm / dark 중 선택 |
| 카드 장수 | 10장 | 고정 (Hook 1 / 본문 7 / 정리 1 / CTA 1) |
| 저장 위치 | `50-my-work/cardnews-maker/` | 고정 |

### 테마 미리보기

| 테마 | 느낌 | 주요 색상 |
|------|------|----------|
| mint | 깔끔하고 시원한 | #00b894, #e0f7f1, #1a1a1a |
| warm | 따뜻하고 친근한 | #e17055, #fff4e6, #2d3436 |
| dark | 세련되고 모던한 | #dfe6e9, #2d3436, #ffffff |

---

## 워크플로우

### Step 0: 이어하기 확인
**타입**: prompt

`50-my-work/cardnews-maker/` 폴더를 확인한다.
- `01-cardnews-plan.md`만 있으면 → "기획서가 있습니다. HTML 생성(Step 3)부터 이어할까요?" 확인
- 두 파일 모두 있으면 → "기존 결과물이 있습니다. 새로 만들까요, 기존 파일을 확인할까요?" 확인
- 파일 없으면 → Step 1로 진행

### Step 1: 주제 입력 + 테마 선택
**타입**: prompt

사용자에게 주제, 테마, 톤을 받는다.

톤 선택:
| 톤 | 특징 | 예시 |
|------|------|------|
| 친근한 (기본) | 반말, 이모지 활용, 대화체 | "이거 모르면 손해야!" |
| 전문적인 | 경어, 데이터 중심, 신뢰감 | "전문가들이 추천하는 방법입니다" |
| 짧고 강한 | 한 줄 임팩트, 키워드 중심 | "딱 하나만 기억하세요" |

입력 검증:
- 주제가 비어있거나 5자 미만이면 → "주제를 한 문장으로 입력해주세요. 예: '혼밥 초보 레시피 5선'" 안내 후 재입력
- 테마 미지정 시 → mint 자동 적용, "테마를 mint로 적용했습니다" 안내
- 테마 오탈자 시 → 허용 테마 목록(mint/warm/dark) 안내 후 재입력
- 톤 미지정 시 → "친근한" 자동 적용

### Step 2: 기획서 생성
**타입**: generate

주제를 바탕으로 10장 카드뉴스 기획서를 생성한다.

카드 구조 (고정):
- 카드 1: Hook — 호기심 유발, 숫자나 질문 활용
- 카드 2~8: 본문 — 핵심 내용 전달, 카드당 2~3줄 이내
- 카드 9: 정리 — 전체 내용 요약, 한눈에 보기
- 카드 10: CTA — 행동 유도 (댓글, 저장, 공유)

각 카드에 포함할 항목:
- 카드 번호, 카드 역할, 제목, 본문 (2~3줄 이내), 시각 요소 설명, 디자인 메모

파일 저장 전 기존 파일 확인:
- `01-cardnews-plan.md`가 이미 있으면 → "기존 기획서를 덮어쓸까요?" 확인
- 확인 후 `50-my-work/cardnews-maker/01-cardnews-plan.md`에 저장

### Step 3: HTML 카드뉴스 생성
**타입**: generate

`01-cardnews-plan.md`를 읽고 HTML 카드뉴스를 생성한다.

의존성 체크:
- `01-cardnews-plan.md`가 없으면 → "기획서가 없습니다. Step 2부터 실행해주세요" 안내 후 중단

HTML 규격:
- 카드 크기: 1080x1080px 정사각
- 배치: 세로 스크롤
- 폰트: Noto Sans KR (Google Fonts CDN)
- 모바일 반응형: max-width 제한 + 비율 유지
- 카드 번호: 우상단에 "n / 10" 표시
- 외부 JS 라이브러리 사용 금지 — 순수 HTML + CSS만
- 이미지 파일 불러오기 금지 — 이모지와 CSS로 시각 표현

테마 적용:
- CSS 변수(`:root`)로 색상 관리
- 선택된 테마의 색상 팔레트를 일괄 적용
- 카드별 포인트 색상은 테마 메인 색상 기반으로 변형

오버플로우 방어:
- 카드당 본문 최대 120자. 초과 시 기획서 단계에서 줄인다
- 제목은 최대 20자. 초과 시 `font-size: clamp(28px, 4vw, 48px)`로 자동 축소
- CSS에 `overflow: hidden` + `text-overflow: ellipsis` 적용하여 레이아웃 깨짐 방지

파일 저장 전 기존 파일 확인:
- `02-cardnews.html`이 이미 있으면 → "기존 HTML을 덮어쓸까요?" 확인
- 확인 후 `50-my-work/cardnews-maker/02-cardnews.html`에 저장

### Step 4: 검토 체크리스트
**타입**: review

생성된 기획서와 HTML을 아래 항목으로 점검한다.

| 항목 | pass | warn | fail |
|------|------|------|------|
| Hook | 숫자·질문·감탄 중 1개 이상 포함 | 호기심 요소가 약함 | 호기심 요소 없음 |
| 본문 분량 | 카드당 80자 이하 | 81~120자 | 121자 이상 |
| 역할 배분 | Hook 1 / 본문 7 / 정리 1 / CTA 1 | - | 구성 불일치 |
| CTA | 구체적 행동 동사 포함 (댓글, 저장, 공유 등) | 행동 유도가 모호 | CTA 없음 |
| 시각 요소 | 모든 카드에 이모지 또는 CSS 시각 표현 | 2장 이하 누락 | 3장 이상 누락 |
| 테마 일관성 | 선택 테마 색상이 전체 일관 | 1~2장 색상 불일치 | 3장 이상 불일치 |
| 모바일 | 540px 이하에서 텍스트 읽힘 | 일부 텍스트 잘림 | 레이아웃 깨짐 |
| 카드 번호 | 모든 카드에 "n / 10" 표기 | - | 번호 누락 |

결과를 pass / warn / fail로 표시하고, warn이나 fail 항목이 있으면 수정 제안을 함께 안내한다. fail이 1개 이상이면 해당 Step을 재실행하도록 권고한다.

### Step 5: 완료 안내
**타입**: prompt

브라우저에서 HTML 파일을 열어 최종 결과를 확인한다.

완료 메시지:
```
카드뉴스 생성 완료!

저장된 파일:
- 기획서: 50-my-work/cardnews-maker/01-cardnews-plan.md
- HTML: 50-my-work/cardnews-maker/02-cardnews.html

다른 주제로 다시 만들려면: /cardnews-maker
```

## 절대 하지 마

- 카드 장수를 10장이 아닌 다른 수로 바꾸지 마
- 특정 주제에 종속된 구조로 만들지 마 — 주제는 항상 사용자 입력값
- 외부 JavaScript 라이브러리를 HTML에 추가하지 마
- 이미지 파일을 외부에서 불러오지 마
- 기획서 없이 HTML을 바로 생성하지 마
- 사용자 파일을 확인 없이 덮어쓰지 마
- 투자 권유, 매수·매도 판단, 수익 보장 표현을 넣지 마
