# Clip 8. 토요일 오프라인 웹앱 과제 안내

## 목표

| 항목 | 내용 |
|---|---|
| 과제 목적 | 작은 웹앱 하나를 기획부터 배포까지 해본다 |
| 오늘과 연결 | 오늘은 스킬, 토요일은 웹앱 |
| 원칙 | 이 클립은 안내만 한다. 웹앱 제작은 토요일에 진행한다 |

## 과제 범위

| 포함 | 제외 |
|---|---|
| 한 화면 MVP | 로그인 |
| 입력과 출력이 명확한 웹앱 | DB |
| 정적 배포 가능한 구조 | 결제 |
| Vercel 또는 GitHub Pages 배포 | 복잡한 백엔드 |

## 진행 순서

| Step | 할 일 | 산출물 |
|---:|---|---|
| 1 | 웹앱 아이디어 한 줄 정하기 | 아이디어 |
| 2 | Claude Code와 기획 대화 | 기능 범위 |
| 3 | `show-me-the-prd`로 PRD 작성 | PRD 문서 |
| 4 | `kkirikkiri`로 구현 팀 구성 | 팀 계획 |
| 5 | Claude Code로 MVP 구현 | 웹앱 코드 |
| 6 | Vercel 또는 GitHub Pages 배포 | 배포 URL |
| 7 | 회고 작성 | 제출 문서 |

## 아이디어 범위 예시

| 유형 | 예시 |
|---|---|
| 계산기 | 비용 계산기, 절감 시간 계산기 |
| 진단기 | 업무 자동화 가능성 진단기 |
| 추천기 | 콘텐츠 주제 추천기 |
| 변환기 | 메모를 카드뉴스 구조로 변환 |
| 대시보드 | 뉴스/지표 요약 대시보드 |
| 체크리스트 | 프로젝트 준비 체크리스트 |

## 과제 프롬프트 1 — 기획 대화

```text
나는 토요일 오프라인 모임에서 만들 작은 웹앱을 기획하려고 해.
내 아이디어는 [아이디어]야.

바로 구현하지 말고,
사용자, 사용 상황, 입력값, 출력값, 핵심 기능 3개,
이번 버전에서 제외할 기능을 먼저 질문해줘.
```

## 과제 프롬프트 2 — PRD 만들기

```text
방금 정리한 웹앱 아이디어를 show-me-the-prd로 PRD 문서로 만들어줘.
토요일 오프라인 모임에서 실제 구현할 수 있게
MVP 범위를 작게 잡아줘.
```

## 과제 프롬프트 3 — kkirikkiri 팀 구성

```text
이 PRD를 바탕으로 kkirikkiri로 웹앱 구현 팀을 구성해줘.

필요한 관점:
- PM
- 프론트엔드 개발자
- UI 디자이너
- QA 리뷰어

각 역할이 맡을 작업과 병렬로 진행할 순서를 정리해줘.
```

## 과제 프롬프트 4 — 구현

```text
PRD와 kkirikkiri 팀 계획을 바탕으로 웹앱을 구현해줘.

이번 버전은 한 화면에서 동작하는 MVP로 만들고,
로그인, DB, 결제, 복잡한 서버 기능은 제외해줘.
```

## 과제 프롬프트 5 — 배포

```text
지금 만든 웹앱을 배포하려고 해.
이 프로젝트 구조를 보고 Vercel과 GitHub Pages 중 더 쉬운 방법을 추천해줘.
그 방법으로 배포 준비 파일과 절차를 안내해줘.
```
