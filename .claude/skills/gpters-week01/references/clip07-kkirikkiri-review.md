# Clip 7. kkirikkiri로 만든 스킬 리뷰하기

## 목표

| 항목 | 내용 |
|---|---|
| 목적 | 만든 스킬을 여러 관점에서 검토한다 |
| 대상 스킬 | `cardnews-maker`, `stock-news-analyst` |
| 핵심 도구 | `kkirikkiri` |
| 결과물 | 스킬별 개선안, 실패 가능성, 다음 수정 방향 |

## 진행 순서

| Step | 할 일 | 산출물 |
|---:|---|---|
| 1 | `cardnews-maker` 리뷰 관점 설정 | 카드뉴스 스킬 리뷰 |
| 2 | 리뷰 결과에서 개선안 3개 선택 | 수정 후보 |
| 3 | `stock-news-analyst` 리뷰 관점 설정 | 주식 뉴스 분석 스킬 리뷰 |
| 4 | 리뷰 결과에서 개선안 3개 선택 | 수정 후보 |
| 5 | 두 스킬 공통 개선 원칙 정리 | 회고 |

## 카드뉴스 스킬 리뷰 프롬프트

```text
cardnews-maker 스킬을 여러 관점에서 검토해줘.

필요한 관점:
- 콘텐츠 기획자
- 카피라이터
- 디자이너
- QA 리뷰어

검토할 것:
- 수강생이 혼자 따라갈 수 있는가
- 카드뉴스 구조가 안정적인가
- 결과물이 너무 길거나 산만해지지 않는가
- 실패했을 때 복구 프롬프트가 충분한가
- 개선안 5개
```

## 주식 뉴스 분석 스킬 리뷰 프롬프트

```text
stock-news-analyst 스킬을 여러 관점에서 검토해줘.

필요한 관점:
- 리서처
- 주식 애널리스트
- 데이터 정리자
- 리스크 검토자

검토할 것:
- 기사 URL 수집 흐름이 이해 가능한가
- 중복/유사 이슈 정리 기준이 명확한가
- 종목 분석 리포트 구조가 실용적인가
- 투자 권유처럼 보이지 않게 안전장치가 있는가
- 개선안 5개
```

## 리뷰 결과 정리 양식

| 스킬 | 가장 큰 문제 | 바로 고칠 것 3개 | 다음에 고칠 것 |
|---|---|---|---|
| `cardnews-maker` |  |  |  |
| `stock-news-analyst` |  |  |  |

## 완료 기준

| 항목 | 완료 조건 |
|---|---|
| 카드뉴스 리뷰 | 리뷰 결과가 있다 |
| 주식 뉴스 분석 리뷰 | 리뷰 결과가 있다 |
| 개선안 선택 | 각 스킬별 수정 후보 3개를 골랐다 |
| 회고 | 공통 개선 원칙을 정리했다 |
