# Clip 6. 주식 뉴스 분석 스킬 만들기

## 목표

| 항목 | 내용 |
|---|---|
| 만들 스킬 | `stock-news-analyst` |
| 목적 | 키워드 기반 뉴스 수집, 중복 정리, 주식 뉴스 분석 리포트 작성 흐름을 스킬화 |
| 첫 실습 키워드 | `하이닉스` |
| 핵심 도구 | `insane-search`, `show-me-the-prd`, `skillers-suda` |

## 산출물

| 파일 | 내용 |
|---|---|
| `01-news-urls.csv` | 기사 제목, 언론사, 시간, URL |
| `02-deduped-news.md` | 중복·유사 이슈 묶음 |
| `03-stock-news-report.md` | 주식 뉴스 분석 리포트 |
| PRD 문서 | 스킬 요구사항 |
| `stock-news-analyst` | 키워드 입력형 스킬 초안 |

## 진행 순서

| Step | 할 일 | 산출물 |
|---:|---|---|
| 1 | 네이버 뉴스에서 `하이닉스` 검색 URL 확인 | 검색 URL |
| 2 | `insane-search`로 기사 URL 수집 흐름 안내 | `01-news-urls.csv` |
| 3 | 기사 목록에서 중복·유사 이슈 묶기 | `02-deduped-news.md` |
| 4 | 종목 뉴스 분석 리포트 작성 | `03-stock-news-report.md` |
| 5 | 흐름을 PRD로 일반화 | PRD 문서 |
| 6 | `skillers-suda`로 스킬 생성 | `stock-news-analyst` |
| 7 | 다른 키워드로 재실행 | 재사용 확인 |

## 네이버 뉴스 검색 URL

| 항목 | 값 |
|---|---|
| 키워드 | `하이닉스` |
| URL | `https://search.naver.com/search.naver?where=news&ie=utf8&sm=nws_hty&query=%ED%95%98%EC%9D%B4%EB%8B%89%EC%8A%A4` |

## 실습 프롬프트 1 — URL 수집

```text
insane-search를 활용해서 아래 네이버 뉴스 검색 결과 페이지에서
최근 기사 URL을 수집하는 방법을 안내해줘.

검색 키워드: 하이닉스
URL: https://search.naver.com/search.naver?where=news&ie=utf8&sm=nws_hty&query=%ED%95%98%EC%9D%B4%EB%8B%89%EC%8A%A4

수집할 정보:
- 기사 제목
- 언론사
- 시간
- 기사 URL
- 핵심 문장

결과는 01-news-urls.csv로 정리하게 해줘.
```

## 실습 프롬프트 2 — 중복·유사 이슈 정리

```text
수집한 기사들을 보고
중복되거나 같은 이슈를 다룬 기사들을 묶어줘.

묶음 기준:
- 같은 사건
- 같은 증권사 리포트
- 같은 실적/목표주가 이슈
- 같은 제품/기술 이슈
- 같은 시장 반응

결과는 02-deduped-news.md로 정리해줘.
```

## 실습 프롬프트 3 — 리포트 작성

```text
정리한 기사 묶음을 바탕으로
주식 뉴스 분석 리포트를 작성해줘.

리포트에는 다음을 포함해줘.
- 핵심 요약 3줄
- 주요 이슈 3~5개
- 종목 관점 해석
- 긍정 요인
- 리스크 요인
- 후속 확인 키워드
- 투자 권유가 아니라는 주의사항

결과는 03-stock-news-report.md로 저장해줘.
```

## 실습 프롬프트 4 — PRD 만들기

```text
방금 진행한 흐름을
사용자가 종목명이나 키워드를 입력하면 반복 실행할 수 있는
stock-news-analyst 스킬로 만들기 위한 PRD로 정리해줘.

show-me-the-prd 형식으로 만들어줘.

중요:
- 하이닉스에 고정하지 말 것
- 사용자가 입력한 종목명 또는 키워드로 동작할 것
- 뉴스 기반 분석이며 투자 권유가 아님을 명시할 것
```

## 실습 프롬프트 5 — 스킬 만들기

```text
이 PRD를 바탕으로 skillers-suda를 사용해서
stock-news-analyst 스킬을 만들어줘.

이 스킬은 하이닉스에 고정되지 않고,
사용자가 입력한 종목명 또는 키워드로 동작해야 해.
```

## 리포트 구조

| 섹션 | 내용 |
|---|---|
| 1. 입력 키워드 | 사용자가 입력한 종목명/키워드 |
| 2. 핵심 요약 | 전체 뉴스 흐름 3줄 요약 |
| 3. 수집 기사 | 제목, 언론사, URL, 핵심 내용 |
| 4. 중복·유사 이슈 | 같은 내용을 다룬 기사 묶음 |
| 5. 핵심 이슈 | 주요 쟁점 3~5개 |
| 6. 종목 관점 해석 | 매출, 이익률, 수급, 밸류에이션, 리스크 영향 |
| 7. 후속 검색 키워드 | 다음에 확인할 키워드 5개 |
| 8. 주의사항 | 뉴스 기반 분석이며 투자 권유가 아님 |

## 완료 기준

| 항목 | 완료 조건 |
|---|---|
| URL 수집 | 기사 URL 목록이 정리됨 |
| 중복 정리 | 유사 이슈별 기사 묶음이 만들어짐 |
| 리포트 | 주식 뉴스 분석 리포트가 작성됨 |
| PRD | 키워드 입력형 스킬 요구사항이 정리됨 |
| 스킬 | `stock-news-analyst` 초안 생성 |
| 재실행 | 다른 키워드로도 사용할 수 있음 |
