# Clip 3. hook 설정하기

## 목표

| 항목 | 내용 |
|---|---|
| 목적 | Claude Code 작업에 알림과 안전장치를 추가한다 |
| 설정 방향 | 완료/확인 필요 알림, 위험 명령 차단 |
| 원칙 | 복잡한 자동화보다 오늘 필요한 최소 hook만 다룬다 |

## hook 개념

| 구분 | 설명 |
|---|---|
| Instruction | Claude에게 지켜달라고 말하는 규칙 |
| Hook | 특정 상황에서 자동으로 실행되는 강제 장치 |
| 오늘의 메시지 | 빠른 실습 모드에는 최소 안전장치가 필요하다 |

## 오늘 설정할 hook

| hook | 목적 |
|---|---|
| 작업 완료/확인 필요 알림 | Claude가 멈췄거나 사용자 확인이 필요할 때 놓치지 않기 |
| 위험 명령 차단 | `rm -rf`, `git reset --hard`, `sudo` 같은 명령을 한 번 막기 |

## 수강생 입력 프롬프트

```text
Claude Code hook을 설정하려고 해.
빠른 실습 모드로 진행할 예정이라 최소 안전장치를 넣고 싶어.

원하는 hook:
1. 작업 완료 또는 확인 필요 시 알림을 주는 hook
2. rm -rf, git reset --hard, sudo 같은 위험 명령을 막는 hook

현재 워크스페이스 기준으로 어떤 파일을 만들거나 수정해야 하는지 먼저 설명해줘.
그다음 내가 따라할 수 있게 단계별로 안내해줘.
마지막에는 테스트 방법도 알려줘.
```

## 확인 프롬프트

```text
방금 설정한 hook이 제대로 동작하는지 테스트하고 싶어.
알림 hook과 위험 명령 차단 hook을 각각 안전하게 테스트하는 방법을 알려줘.
```

| 알림 hook 목적을 이해했다 |  |
| 위험 명령 차단 hook 목적을 이해했다 |  |
| 설정 후 테스트 방법을 확인했다 |  |
