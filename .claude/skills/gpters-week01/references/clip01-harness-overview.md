# Clip 1. 하네스 엔지니어링 개념 정리

## 목표

| 항목 | 내용 |
|---|---|
| 핵심 개념 | Claude Code를 내가 의도한 방식으로 동작하게 만드는 작업 환경 설계 |
| 강조점 | 모델 자체보다 모델이 일하는 구조가 결과를 크게 바꾼다 |
| 오늘 연결 | 스킬 제작 전에 Claude Code가 일할 환경부터 정리한다 |

## 한 줄 정의

```text
하네스 엔지니어링은 모델에게 뭘 말할지가 아니라,
모델이 내가 원하는 방식으로 일하게 만들려면
어떤 작업 환경, 규칙, 안전장치, 반복 흐름을 둘지 설계하는 일입니다.
```

## 핵심 구성요소

| 구성요소 | 의미 |
|---|---|
| Claude 모델 | 추론과 생성 담당 |
| 작업 규칙 | 어떤 순서와 기준으로 일할지 정함 |
| 폴더/파일 구조 | 결과물이 저장될 위치를 고정 |
| 안전장치 | 위험한 명령이나 설정 변경 전에 확인 |

## 핵심 설명

| 질문 | 답 |
|---|---|
| 왜 하네스가 필요한가 | Claude 성능이 좋아도 작업 순서, 권한, 안전장치, 반복 흐름이 없으면 매번 흔들린다 |
| 무엇을 바꾸는가 | 모델 자체가 아니라 모델이 일하는 환경을 바꾼다 |
| 좋은 하네스란 | 같은 작업을 다음에도 안정적으로 다시 하게 만드는 구조 |
| 오늘의 목적 | Claude Code가 수강생 의도대로 움직이도록 기본 작업 환경을 만든다 |

## 실습 안내

```text
Claude Code를 잘 쓰려면 좋은 질문만으로는 부족합니다.
어떤 파일을 읽을지, 어떤 순서로 작업할지, 결과를 어디에 저장할지,
위험한 작업은 언제 확인받을지 정해야 합니다.

이런 작업 규칙과 환경을 설계하는 것이 하네스 엔지니어링입니다.
```

## 오늘 실습과 연결

| 오늘 할 일 | 하네스 관점 |
|---|---|
| 실행 명령 정리 | 매번 긴 명령을 치지 않게 한다 |
| 작업 모드 이해 | 실습 흐름이 끊기지 않게 한다 |
| hook 설정 | 위험한 행동과 놓치는 알림을 줄인다 |
| plugin 설치 | 필요한 능력을 작업 환경에 추가한다 |
| skill 제작 | 반복 업무 흐름을 다시 실행 가능하게 만든다 |
| kkirikkiri 리뷰 | 만든 흐름의 문제점을 리뷰로 확인한다 |
