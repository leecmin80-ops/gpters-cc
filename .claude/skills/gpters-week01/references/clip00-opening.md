# Clip 0. 오프닝

## 목표

| 항목 | 내용 |
|---|---|
| 오늘의 주제 | 반복 가능한 업무 흐름을 Claude Code 스킬로 만들기 |
| 핵심 내용 | 스킬은 프롬프트 모음이 아니라 다시 실행 가능한 작업 매뉴얼 |
| 실습 방식 | 만들기 → 써보기 → 리뷰받기 → 개선하기 |

## 결과물

| 산출물 | 설명 |
|---|---|
| `cardnews-maker` | 주제/자료를 카드뉴스 기획서와 HTML 카드뉴스로 바꾸는 스킬 |
| `stock-news-analyst` | 종목명/키워드로 뉴스 URL을 모으고 주식 뉴스 분석 리포트를 쓰는 스킬 |
| kkirikkiri 리뷰 | 두 스킬의 개선점을 정리한 리뷰 결과 |

## 진행 순서

| 순서 | 내용 |
|---:|---|
| 1 | Claude Code가 내 의도대로 움직이도록 하네스 개념을 잡는다 |
| 2 | alias와 실행 모드, hook으로 실습 환경을 정리한다 |
| 3 | GPTaku Plugin을 설치한다 |
| 4 | 카드뉴스 제작 흐름을 스킬로 만든다 |
| 5 | 주식 뉴스 분석 흐름을 스킬로 만든다 |
| 6 | kkirikkiri로 두 스킬을 리뷰한다 |
| 7 | 토요일 오프라인 웹앱 과제를 안내한다 |

## 실습 안내

```text
오늘 목표는 Claude Code로 앱 하나를 만드는 게 아닙니다.
반복 가능한 작업 흐름을 스킬로 만드는 겁니다.

스킬은 프롬프트 모음이 아닙니다.
내가 반복해서 하는 업무 흐름을 Claude Code가 다시 실행할 수 있게 만든 작업 매뉴얼입니다.

오늘은 콘텐츠 제작 흐름 하나, 리서치/분석 흐름 하나를 스킬로 만들고
마지막에는 kkirikkiri로 개선점을 정리합니다.
```

## 완료 기준

| 항목 | 확인 |
|---|---|
| 오늘 산출물 3종을 이해했다 | `cardnews-maker`, `stock-news-analyst`, kkirikkiri 리뷰 |
| 오늘 진행 순서를 이해했다 | 환경 세팅 → 플러그인 설치 → 스킬 제작 → 리뷰 |
